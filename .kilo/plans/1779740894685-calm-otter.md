# Plan: Implement Audio Chunking for Groq Transcription

## Goal
Modify the Groq transcription logic to handle audio files larger than the 25MB free tier limit by splitting them into smaller chunks and merging the results.

## Analysis
- **Constraint**: Groq's free tier has a 25MB limit per audio request.
- **Current Implementation**: `main.py:transcribe_with_groq` extracts audio to a single `.wav` file and uploads it entirely.
- **Audio Math**: 16kHz mono PCM WAV $\approx$ 352.8 KB per minute.
  - 25MB $\approx$ 70 minutes of audio.
  - However, to be safe and allow for overhead/different formats, chunks of 10-15 minutes are recommended.
- **Target**: Ensure the pipeline can handle videos of 1+ hours using Groq.

## Implementation Steps

### 1. Audio Chunking Logic
- Create a helper function `split_audio_into_chunks(audio_path, chunk_duration_sec=600)`:
  - Use FFmpeg to split the audio file into segments of `chunk_duration_sec` (e.g., 10 minutes).
  - Use the `segment` muxer to ensure precise splitting without re-encoding.
  - Return a list of paths to the generated chunk files.

### 2. Update `transcribe_with_groq` in `main.py`
- Modify the flow to:
  1. Extract audio to a temporary file as currently done.
  2. Check the file size of the extracted audio.
  3. If size $> 20\text{MB}$:
     - Call `split_audio_into_chunks`.
     - For each chunk:
       - Send to Groq API.
       - Offset the `start` and `end` timestamps of segments and words by the cumulative duration of previous chunks.
       - Append results to a master list.
  4. If size $\le 20\text{MB}$:
     - Process as a single request (current behavior).
  5. Merge all processed segments and words into the final transcript object.
  6. Clean up all temporary chunk files.

### 3. Timestamp Alignment
- Maintain a `cumulative_offset` variable.
- For each chunk $i$, add `cumulative_offset` to every timestamp in the response.
- Update `cumulative_offset` by adding the actual duration of chunk $i$.

### 4. Verification
- Test with a video file whose audio exceeds 25MB (e.g., a 60-minute video).
- Verify that the final transcript is continuous and timestamps align correctly with the original video.
- Compare results with `faster-whisper` to ensure no significant loss in quality/accuracy at chunk boundaries.

## Success Criteria
- Videos longer than 1 hour are successfully transcribed via Groq without `413 Request Entity Too Large` errors.
- Timestamps in the final merged transcript are accurate.
- Temporary files are cleaned up properly.
